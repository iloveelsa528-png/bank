# 백로그

## 7단계 — 학교 기출 패턴 기반 문제 생성 (미구현)

### 개요
업로드된 시험지를 특정 학교의 기출문제로 간주하고, 기출 출제 스타일을 추출하여
새로운 지문에 동일한 스타일로 문제를 생성하는 기능.

단순 유사 문제 생성(4단계)과의 차이점:
- 4단계: 같은 지문 기반으로 유사/변형 문제 생성
- 7단계: 특정 학교의 출제 패턴을 추출 → 새 지문에 그 패턴 적용

### 요구사항

#### 7-1. 기출 패턴 추출
업로드된 시험지(기출문제)에서 아래 항목을 추출하여 저장:
- 문제 유형 (내용 이해, 추론, 적용, 표현상 특징 등)
- 발문 방식 (윗글을 읽고 / ~에 대한 설명으로 / ~의 이유로 적절한 것은)
- 선택지 구성 방식 (서술 길이, 구조, 오답 패턴)
- 출제 의도 및 난이도 분포
- 오답 유형 패턴 (함정 선지 구성 방식)

#### 7-2. 새 지문 입력
분석이 완료된 기출 패턴을 기반으로, 아래 중 하나를 새 지문으로 입력:
- 교과서 본문
- 새 문학 작품 (시, 소설, 수필 등)
- 새 독서 지문 (논설문, 설명문 등)

#### 7-3. 기출 스타일 적용 문제 생성
추출된 기출 패턴을 새 지문에 그대로 적용하여 문제 생성:
- 기출 발문 방식 유지
- 기출 선택지 구성 방식 유지
- 기출 난이도 분포 유지
- 기출 오답 패턴 유지

#### 7-4. 저장 및 관리
생성된 문제를 문제은행에 저장:
- 기출 출처(학교명, 연도, 시험 종류) 태그
- 기출 패턴 프리셋 저장 및 재사용

### 구현 시 고려사항
- 기출 패턴은 DB에 별도 테이블로 저장 (예: `exam_patterns`)
- 한 학교의 기출을 여러 개 누적하면 패턴 신뢰도 향상 가능
- 기존 1~5단계 흐름과 독립적으로 동작하거나 4단계 이후 분기 가능

### 관련 DB 테이블 (예시)
```sql
create table exam_patterns (
  id uuid default gen_random_uuid() primary key,
  school_name text,
  exam_year text,
  exam_type text,  -- 중간고사, 기말고사, 모의고사 등
  question_types jsonb,
  stem_patterns jsonb,
  choice_patterns jsonb,
  difficulty_distribution jsonb,
  wrong_answer_patterns jsonb,
  source_problem_id uuid references problems(id),
  created_at timestamptz default now()
);
```

---
*현재 구현 범위: 1~6단계 완료. 7단계는 위 요구사항 확정 후 구현 예정.*
